FlakBoy

Ce programme est le résultat du projet de fin de 1ère année de DUT. Nous étions 2 étudiants : Bady et Yan.

Pour lancer le programme, télécharger "FlakBoy_runnable.jar", ouvrez une console et tapez java -jar FlakBoy_runnable.jar

Présentation

FlakBoy est à la base un jeu Flash, que vous pourrez tester ici. Le but est donc d'infliger à un personnage le plus de dégâts possibles, grâce à différentes armes. L'intérêt était ici de porter le jeu dans un langage que nous connaissions, le Java. Bien que le principe reste le même, le jeu fut simplifié :

  • Formes graphiques simples (2D, le personnage est représenté par un rond, les armes sont des carrés).
  • Emplacement fixe du personnage au début.
  • 4 armes : Eksasaute (Ressort), Mine, Tourelle, Piques.
  • Pas de niveaux.

How to play

Le rond rouge représente le personnage. Les lignes bleues sont les zones où les armes peuvent être posées. Pour poser une arme, cliquez sur l'une des armes à droite, puis cliquez ensuite sur l'une des zones bleues. Pour vendre une arme, cliquez sur "Vendre une arme", puis cliquez sur l'arme que vous souhaitez vendre. Pour commencer une partie, cliquer sur Go.

Les armes

Eksasaute

Cette arme, comparable à un ressort est une arme fixe. Lorsque le personnage arrive dessus, il est renvoyé avec une vitesse "moyenne". Le personnage peut tomber 4 fois dessus, après l'arme disparait.

Mine

Arme fixe. Lorsque le personnage arrive dessus, il est renvoyé avec une vitesse "forte". L'arme disparait aussitôt que le personnage passe dessus.

Piques

Arme déclenchable. Lorsque le personnage passe dans la zone d'activation, l'arme s'élargit et ralentit considérablement le personnage, jusqu'à ce qu'il quitte la zone d'activation. On peut comparer son effet à celui d'un "mur gluant". L'arme n'a pas de nombre d'utilisations limité.

Tourelle

Arme déclenchable. Lorsque le personnage passe dans la zone d'activation, des missiles sont lancés jusqu'à ce qu'il quitte la zone d'activation. L'arme n'a pas de limite de nombre d'utilisations limité.
